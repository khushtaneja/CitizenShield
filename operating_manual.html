OPERATING MANUAL

Suspicious Activity Reporting System (CitizenShield)

Domain: National Security and Defence Technologies Version: 2.0 Date: April 2026

1. System Overview

CitizenShield is a dual-portal web application designed to strengthen national security through community participation. It allows citizens to report suspicious incidents and provides authorized authorities with a centralized dashboard to track, verify, and resolve these threats.

Key Components:

  • Citizen Portal: Public-facing site for reporting.
  • Admin Dashboard: Restricted panel for law enforcement/analysts.
  • Backend Server: Node.js Express server for persistent data storage.
  • Offline Fallback: Automatic switch to localStorage if the server is offline.

2. Technical Setup & Installation

To run the full version of the system with persistent data, you must set up the backend server.

2.1 Prerequisites

  • Node.js: Must be installed (v14 or higher recommended).
  • Web Browser: Chrome, Edge, or Firefox.

2.2 First-Time Setup

  1. Open your terminal/command prompt.
  2. Navigate to the backend folder: 
    cd "c:/MY ART/web app/backend"
  3. Install dependencies: 
    npm install

2.3 Starting the System

You have two ways to start the backend:

  • Windows (Easy): Double-click the start-server.bat file in the backend folder.
  • Terminal: Run npm start inside the backend directory.
IMPORTANT: The backend runs on http://localhost:5000. Keep the terminal window open while using the application. If you close it, the app will switch to "Offline Mode".

3. Citizen User Guide (Reporting)

3.1 Filing a Report

  1. Open index.html in your browser.
  2. Navigate to "Report Activity".
  3. Fill in the required fields:
    • Full Name: Your identity (handled discreetly).
    • Location: Specific details of where the incident occurred.
    • Description: Factual observation of the suspicious behavior (min. 20 chars).
    • Image: Optional photographic evidence.
  4. Click Submit Report.

3.2 Submission Confirmation

Once submitted, the system will provide a Report ID (e.g., RPT-K9X2J). Keep this ID for your records.

4. Admin User Guide (Management)

4.1 Accessing the Dashboard

  1. Navigate to the Admin Dashboard link in the global navigation bar.
  2. Enter the authorized credentials:
    • Username: admin
    • Password: admin123

4.2 Managing Reports

The dashboard provides a real-time feed of all incoming threats.

Action Description
Filter Use the status buttons (Pending, Verified, Resolved) to sort the feed.
Search Filter by location or name using the search bar.
Status Update Use "Mark Verified" when confirmed, or "Mark Resolved" after intervention.
Delete Remove duplicate or false reports permanently.

4.3 Demo Mode

If the dashboard is empty during a presentation, click "Load Demo Reports" at the bottom to simulate a busy operations environment.

5. Data Flow & Security

5.1 Where is the Data?

  • Online Mode: Data is saved in backend/data/reports.json. This is persistent and shared across all devices.
  • Offline Mode: Data is saved in the browser's localStorage.

5.2 Security Measures

  • Data Validation: Prevents SQL injection and empty/invalid data entry.
  • Admin Authentication: Restricts dashboard access to personnel with valid credentials.
  • Encapsulation: The backend separates logic from presentation for better security.

6. Troubleshooting

"The website says 'Server Offline'"
Ensure you ran npm start in the backend directory. The app will fallback to local storage automatically.

"Images are not uploading"
Ensure the file size is under 5MB.

"I forgot the Admin Password"
For the current prototype, credentials are hardcoded as admin / admin123.

Presentation Tip: During your assessment, show the backend/data/reports.json file to your professor after submitting a report. This proves the system is successfully writing data to a persistent database.
